# Real Hardware Data Next Steps

This checklist turns the Smart Energy Monitoring System from a simulation-first repository into a measured embedded-systems portfolio project.

## Goal

Capture real voltage/current readings from a low-voltage DC load using an Arduino or ESP32 with an INA219 current/voltage sensor.

## Evidence to Capture

- Photos of the Arduino/ESP32, INA219 module, wiring, breadboard, and load.
- Raw serial CSV output from the logger.
- Cleaned CSV generated by the Python analysis script.
- Regenerated SVG plots from measured data.
- A short safety note confirming that only low-voltage DC loads were measured.

## Minimum Hardware Setup

- Arduino Uno/Nano or ESP32.
- INA219 current/voltage sensor module.
- Low-voltage DC load such as LED strip, fan, resistor, or small motor.
- USB serial connection for logging.
- Optional multimeter for calibration comparison.

## Minimum Dataset

Create or update these files:

```text
data/raw_measurements.csv
data/cleaned_measurements.csv
analysis/analysis_summary.txt
figures/voltage_current_over_time.svg
figures/power_over_time.svg
figures/cumulative_energy.svg
figures/load_comparison.svg
hardware/prototype_photos/
```

Suggested raw serial format:

```text
timestamp_ms,voltage_v,current_ma,power_mw
```

Suggested cleaned format:

```text
timestamp_s,voltage_v,current_a,power_w,energy_j,load_type,measurement_status,notes
```

Use `measurement_status = measured` for real hardware captures. Keep simulated rows clearly labeled as `simulated`.

## Portfolio Upgrade Criteria

The project becomes portfolio-strong when the README can say:

```text
Captured real INA219 sensor readings from low-voltage DC loads, cleaned the serial data with Python, calculated cumulative energy, and compared measured load behavior through generated plots.
```
